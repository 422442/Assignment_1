1. Is it correct to say that "data is objective, but information is subjective"?

Yes, the statement that "data is objective, but information is subjective" is largely correct. To understand why, we must distinguish between data, facts, and information.

  • Data refers to raw, unorganized characters, numbers, or symbols without context. By itself, it is objective and has no inherent meaning. For example: The numbers 37, 38, 36.5.
  • Facts are pieces of data that have been verified as true and are universally accepted. Like data, facts are objective. For example: A patient's recorded body temperatures are 37°C, 38°C, and 36.5°C.
  • Information is created when data is processed, organized, and given context to make it meaningful. This is where subjectivity enters, as the interpretation can vary. For example: Based on the temperature data, one doctor might conclude, "The patient has a slight fever," while another might say, "The patient's temperature is fluctuating."

Therefore, while the raw data points are objective, the meaning and insights (information) we derive from them can be subjective.

2. How can the challenges of Volume, Velocity, and Variety be extended to include Veracity and Value?

The "3 Vs" can be extended to the "5 Vs" to provide a more complete picture of big data challenges:

  • Veracity: This refers to the quality, accuracy, and trustworthiness of data. Low-quality data leads to poor conclusions. For example, a marketing campaign will fail if the customer database has incorrect email addresses (low veracity).
  • Value: This refers to the usefulness of data. Data only has value if it can be turned into actionable insights that benefit the organization. For example, collecting customer purchase data only has value if it's used to create personalized recommendations that increase sales.

3. Discuss the five key impacting areas for maximizing profitability: Economy, Trust, Regulation, Capitalization, and Clients.

  • Economy: The economic environment dictates investment priorities. In a recession, a company uses data for cost-cutting, not expansion.
  • Trust: Handling data ethically and securely is crucial. A data breach destroys customer trust and revenue.
  • Regulation: Compliance with data laws like GDPR is mandatory. Non-compliance leads to heavy fines.
  • Capitalization: The financial investment in data infrastructure and talent. A large bank can afford an on-premise warehouse, while a startup may use cheaper cloud services.
  • Clients: Client needs should drive all data efforts. Netflix analyzes viewing habits to personalize content, which increases subscriber retention.

4. Analyze: "descriptive and diagnostic analytics explain the past, while predictive and prescriptive analytics shape the future."

This statement is correct. The four types of analytics represent a progression from hindsight to foresight.

  • Descriptive (What happened?) & Diagnostic (Why?): These are retrospective. They analyze historical data to understand past events. Example: A report shows sales dropped (descriptive), and analysis reveals it was due to a competitor's discount (diagnostic).
  • Predictive (What will happen?) & Prescriptive (What to do?): These are forward-looking. They use past data to forecast and influence future events. Example: Forecasting future sales based on trends (predictive) and then recommending an optimal pricing strategy to boost those sales (prescriptive).

5. If a company already has a Data Warehouse, does it really need a Data Lake?

Yes, a company might still need a Data Lake because they serve different, complementary purposes.

  • A Data Warehouse stores structured, processed data for business intelligence and reporting (e.g., weekly sales reports).
  • A Data Lake stores vast amounts of raw, unstructured data (e.g., social media feeds, server logs) for data exploration and machine learning.

They are not mutually exclusive. A Data Warehouse is for known questions and structured reporting by business users, while a Data Lake is for data scientists to discover new insights from raw, diverse datasets. Many modern data architectures use both together.
